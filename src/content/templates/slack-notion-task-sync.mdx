## 이 워크플로우는

Slack에서 특정 이모지(:memo:)가 달린 메시지를 자동으로 Notion 할일 데이터베이스에 추가합니다.
메시지 내용이 할일 제목이 되고, 작성자가 담당자로 설정됩니다.

Slack에서 할일을 직접 노션으로 옮기는 수작업을 완전히 제거하여,
대화 중 생긴 할일을 놓치지 않고 관리할 수 있습니다.

## 주요 기능

- **이모지 트리거**: :memo: 이모지가 달리면 자동으로 할일 생성
- **메시지 내용 추출**: Slack API로 원본 메시지 텍스트 조회
- **Notion 할일 추가**: 제목, 담당자, 상태(To Do) 자동 설정
- **확인 알림**: 노션 추가 완료 후 Slack 스레드에 알림

## 설정 가이드

1. JSON 파일을 n8n에 import합니다
2. Slack OAuth 인증을 설정하고, Slack Bot Token을 `SLACK_BOT_TOKEN` 환경변수에 등록합니다
3. Notion API 인증을 설정하고, 할일 데이터베이스 ID를 입력합니다
   - Notion 데이터베이스 속성: 제목(Title), 담당자(Rich text), 상태(Select: To Do/In Progress/Done)
4. Slack Event Subscription에서 Webhook URL을 Request URL로 설정합니다
   - 구독 이벤트: `reaction_added`
5. 테스트: 아무 메시지에 :memo: 이모지를 추가하고 Notion에 할일이 생성되는지 확인합니다
